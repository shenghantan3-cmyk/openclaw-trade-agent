# OpenClaw 量化交易 Agent 部署文档

## 一、环境准备
### 1. 系统要求
- 操作系统：Linux (CentOS/Ubuntu/Debian 均可)
- Python 版本：3.8+
- Node.js 版本：18+ (用于安装 OpenClaw)

### 2. 依赖安装
```bash
# 安装 Python 依赖
pip install python-dotenv requests pandas

# 安装 OpenClaw (国内镜像)
npm config set registry https://registry.npmmirror.com
npm install -g openclaw --force
```

## 二、OpenClaw 部署
### 1. 初始化配置
```bash
# 初始化 OpenClaw 工作目录
openclaw init

# 切换到工作目录
cd ~/.openclaw/workspace
```

### 2. 配置国内大模型后端
编辑 `~/.openclaw/config.yaml`，添加模型配置：
```yaml
models:
  default:
    provider: volcengine
    model: ark-code-latest
    api_key: "你的火山引擎APIKey"
    base_url: "https://ark.cn-beijing.volces.com/api/v3"
```

### 3. 启动 Gateway 服务
```bash
# 启动 Gateway 后台服务
openclaw gateway start

# 验证服务启动
curl http://localhost:18789/health
```
正常返回：`{"status":"ok"}`

### 4. 功能验证
```bash
# 验证模型调用
openclaw ping

# 正常返回：
# 📡 OpenClaw 网关运行正常
# 🤖 模型调用成功：你好！我是 OpenClaw 助手。
```

## 三、交易插件部署
### 1. 插件安装
将 `generate_trade_order.py` 和 `trade_agent.py` 放到 OpenClaw 插件目录：
```bash
mkdir -p ~/.openclaw/plugins/trade-order
cp generate_trade_order.py ~/.openclaw/plugins/trade-order/
cp trade_agent.py ~/.openclaw/plugins/trade-order/
```

### 2. 插件注册
编辑 `~/.openclaw/config.yaml`，添加插件配置：
```yaml
plugins:
  trade-order:
    path: ~/.openclaw/plugins/trade-order
    entry: generate_trade_order.generate_trade_order
    description: 量化交易订单生成插件
```

### 3. 创建工作目录
```bash
# 创建信号监听目录和日志目录
mkdir -p /opt/openclaw/signals
mkdir -p /opt/openclaw/logs
```

## 四、Agent 启动
### 1. 后台运行 Agent
```bash
# 启动交易Agent，后台运行
nohup python3 ~/.openclaw/plugins/trade-order/trade_agent.py > /var/log/trade-agent.log 2>&1 &

# 查看运行状态
ps aux | grep trade_agent.py
```

### 2. 测试验证
```bash
# 写入测试信号
cat > /opt/openclaw/signals/test_signal.json << EOF
{
    "signal_type": "time_series",
    "strategy_id": "STR001",
    "action": "add",
    "signal_time": "2026-03-13 14:30:00",
    "ticker": "600000.SH",
    "current_position": 800,
    "target_position": 1000
}
EOF

# 查看日志
tail -f /opt/openclaw/logs/order_log.csv
```
正常会生成一条订单记录。

## 五、目录结构
```
/opt/openclaw/
├── signals/          # 信号文件目录，监听新增/更新
│   ├── cross_section_signal.json
│   └── time_series_signal.json
└── logs/             # 日志目录
    └── order_log.csv # 订单生成日志

~/.openclaw/
├── config.yaml       # OpenClaw 配置文件
└── plugins/
    └── trade-order/  # 交易插件目录
        ├── generate_trade_order.py
        └── trade_agent.py
```

## 六、常见问题
### 1. Gateway 启动失败
- 检查18789端口是否被占用：`lsof -i:18789`
- 查看日志：`openclaw gateway logs`

### 2. 模型调用失败
- 检查API Key是否正确
- 检查网络是否能访问模型服务地址

### 3. 信号处理不生效
- 检查信号文件是否为合法JSON格式
- 查看Agent日志：`tail -f /var/log/trade-agent.log`
